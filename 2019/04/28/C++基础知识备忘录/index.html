C++虚析构函数总结

虽然刚开始看的时候被

这一行迷惑了,还以为callq address这种的调用没有使用虚函数表.但是后面用clang++ -S xuxigou.cc分析的时候发现还是有的.

总之,虚析构函数在这种情况下是必须有的.delete pb;虽然delete的是基类的指针,但是由于有虚函数表的存在,所以调用析构函数的时候会去虚函数表里面查询,(基类有一个虚函数表,派生类也有一个虚函数表),这个pb的类的实例指向的是派生类的对象,所以pb去查的虚函数表是派生类的虚函数表吧???然后在派生类的虚函数表里面查到析构函数,虽然名字不同,但是估计被内部处理过(见下面的图),然后就是正确的顺序出现了.先调用的派生类的析构函数,然后再调用基类的析构函数.

具体的内部处理查到effective c++ Item 7上面是这么说的,所以派生类的析构函数因为编译器给它创建了对基类的析构函数的调用动作,所以就可以这么做.

但是如果析构函数里面没有virtual关键词,那么在这个例子里面就没有这两张虚函数表.(因为没有其他的虚函数了),所以调用delete pb的时候去找析构函数,这时没有查表,然后delete内部认为它是一个基类的指针,所以去调用的基类的析构函数.(这种情况下delete pointer,delete一个指针,内部是不会去管有没有继承关系的.)

C++常量折叠问题的理解

参考https://blog.csdn.net/misayaaaaa/article/details/69432679

在C++中，const变量(即常量)值放在编译器的符号表中，计算时编译器直接从表中取值，而不是访问内存，省去了访问内存的时间，这也就是所谓的编译器优化。

隐藏和覆盖的区别

隐藏经常是瞎命名导致的,起一堆一样的名字,又在嵌套的不同作用域.可以是函数,也可以是变量.

而覆盖是与虚函数相关的概念.属于正常操作.而且在C++11里面为了避免这些麻烦,直接出了一个override关键词…

重载必须在同一作用域,在不同作用域中无法重载函数.!!!

如果不在同一作用域,那就变成隐藏了…例如:
